# 랜딩 페이지 스크린샷

이 폴더에 넣는 PNG는 `lingo.jiny.shop`의 랜딩 페이지에서 자동으로 표시됩니다.

## 어떤 파일을 두면 되나

| 파일명 | 노출 위치 | 권장 사이즈 |
|---|---|---|
| `hero.png` | 메인 hero 섹션의 오른쪽 phone mockup 안 | 1080 × 2340 (9:19.5) |
| `01.png` | "실제 화면 미리보기" 섹션 첫 번째 칸 | 1080 × 2340 |
| `02.png` | 두 번째 칸 (단어장 / 활용형 사전 추천) | 1080 × 2340 |
| `03.png` | 세 번째 칸 (복습 / 퀴즈 추천) | 1080 × 2340 |
| `04.png` | 네 번째 칸 (알림 / 학습 통계 추천) | 1080 × 2340 |

## 어떻게 적용되나

- 파일을 이 폴더에 두고 commit + push 하면 docker build 시 함께 컨테이너에 복사됨
- 서버 main.ts의 `useStaticAssets`가 `/screenshots/...` 경로로 서빙
- HTML에서 `<img onerror>`로 폴백 처리 — 파일이 없으면 placeholder 박스

## 파일 없을 때

placeholder 박스가 자동 표시 — "screenshots/01.png 두면 표시" 식 안내.

## 형식

- PNG 권장 (배경 그라데이션과 자연스럽게 어울림)
- JPG도 가능 — 다만 HTML의 `src` 확장자를 맞춰서 같이 수정 필요
- 1080px 폭이면 retina 디스플레이에서도 또렷

## 운영 팁

- iPhone 14/15 등의 시뮬레이터에서 캡처 후 그대로 사용 가능
- 노치/Dynamic Island 그대로 두면 더 실감
- 핵심 정보가 잘 보이는 화면 위주 (오늘 문장 / 단어장 / 복습 / 위젯)
